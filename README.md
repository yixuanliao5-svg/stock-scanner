# Alpha Hunter 2.0 — 自動更新版

台股 × 美股波段掃描 + 隔日當沖觀察清單。GitHub Actions 每天收盤後自動抓資料、重算點位、更新網站。

## 安裝（一次設定，之後全自動）

1. 把這個資料夾的**全部內容**（含隱藏的 `.github` 資料夾）上傳到你的 `alpha-hunter` repo，取代舊檔案。
   - 用 git：`git add -A && git commit -m "v2 auto-update" && git push`
   - 用網頁上傳：注意 `.github/workflows/scan.yml` 的路徑要保留
2. Repo → **Settings → Actions → General → Workflow permissions** → 勾選 **Read and write permissions** → Save
3. Repo → **Actions** 頁籤 → 選 `daily-scan` → **Run workflow** 手動跑第一次（建立全市場歷史資料）
4. 完成。之後每個交易日 14:10（台股收盤後）與 05:10（美股收盤後）自動更新

## 排程說明

- 台股：TWSE OpenAPI 全市場日線（約 1,000+ 檔普通股），當沖清單由此全市場篩選
- 美股：stooq 日線，追蹤清單 10 檔（`scanner.py` 頂部 `US_WATCH` 可自行修改）
- 歷史資料存在 `history/`，自動維護 130 個交易日
- **注意：全市場量比、均線需要至少 21 個交易日的歷史**。第一次啟用後，當沖清單約需 3–4 週才會達到完整準確度（首月量比以可得天數計算，會偏不準）

## 即時報價（富果）

網頁內建「🔄 更新即時價」按鈕與 60 秒自動更新（僅台股）。點位卡會即時標示：在進場區內 / 未達 / 已越過（勿追）/ 已破停損（訊號作廢）。

⚠️ 富果 API key 寫在 index.html 裡（公開）。這是唯讀行情 key、不能下單，但任何人都能用它消耗你的免費額度（60 次/分）。若即時價常失靈，多半是額度被用光——去富果後台重發一把 key，替換 index.html 裡的 `FUGLE_KEY` 即可。

## 手動維護的部分（規則式系統做不到的）

- `scanner.py` 的 `US_EARNINGS`：美股財報日要季度更新一次
- 大盤紅綠燈是機械判定（指數 vs MA5/MA20），重大事件（如 2026-07-17 史上最大跌點）當天請自行加重警戒
- 點位是規則算出來的，不含人為判讀。訊號品質請自行以交易日誌驗證

## 免責

非投資建議。掃描規則未經回測，勝率與期望值未知。每筆交易記錄、30 筆後自行統計，再決定信任程度。
